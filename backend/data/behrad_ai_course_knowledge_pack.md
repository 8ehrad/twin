## Curriculum overview

Ed Donner's AI curriculum is a six-course roadmap covering AI builders, coding agents, AI leadership, LLM engineering, agentic AI engineering and AI systems in production. The public curriculum page describes the six courses as complementary and positions completion of all six, combined with real-world projects, as preparation for AI Engineer work.

The six courses are:

1. **AI Builder: Create Agents, Voice Agents & Automations in n8n**
2. **AI Coder: Complete Claude Code & Coding Agents Course**
3. **AI Leader: Generative AI & Agentic AI for Leaders & Founders**
4. **AI Engineer Core Track: LLM Engineering, RAG, QLoRA, Agents**
5. **AI Engineer Agentic Track: The Complete Agent & MCP Course**
6. **AI Engineer Production Track: Deploy LLMs & Agents at Scale**

The curriculum has two useful high-level tracks:

- **Use products to build agents:** no-code/low-code and automation-heavy AI systems, mostly represented by AI Builder.
- **Use agents to build products:** coding agents, LLM engineering, autonomous agents, and production deployment, represented by AI Coder, Core, Agentic and Production tracks.

For Behrad's digital twin, the key message is:

> Behrad understands the full modern AI delivery chain: LLM application design, retrieval, fine-tuning, agentic workflows, multi-agent orchestration, coding-agent-assisted development, cloud deployment, monitoring and production reliability.

# Course 1: AI Builder - Create Agents, Voice Agents & Automations in n8n

## Short description

This course focused on building useful AI agents and automations quickly using low-code workflows. The emphasis was on turning AI into operational tools: agents that can call APIs, use business data, trigger workflows, answer questions from a knowledge base, interact with users, and automate sales or operational tasks.

For Behrad, the main value was not merely learning a visual automation tool. The value was learning how AI systems are assembled into workflows: triggers, tools, APIs, structured data, retrieval, memory-like context, human handoff, notifications, and business outcomes.

## Main material covered

The course covered:

- Building low-code AI agents for practical business tasks
- Connecting AI agents to external tools and APIs
- Using webhooks and workflow triggers
- Working with structured workflow data, especially JSON
- Creating automations that read from and write to business systems
- Building RAG-backed knowledge base agents
- Creating voice agents that can speak to users and trigger backend workflows
- Integrating AI agents with messaging, email, spreadsheets, CRM-style tools and notification systems
- Using local/self-hosted automation where appropriate
- Understanding how MCP can connect agents to external capabilities
- Thinking about AI agents as business process components, not just chatbots
